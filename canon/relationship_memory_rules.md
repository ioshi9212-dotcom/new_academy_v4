# Relationship Memory Rules

Этот файл описывает, как хранить и использовать отношения между персонажами.

Цифры отношений сами по себе недостаточны.

`affection`, `trust`, `tension`, `jealousy`, `respect`, `curiosity`, `resentment` показывают направление, но не объясняют, почему персонажи ведут себя именно так.

Поэтому для важных пар нужно хранить не только score, но и живую память отношений.

## Базовые поля

Для каждой значимой пары можно использовать:

```txt
pair_id: akira__haru_foster

affection: 0
trust: 0
tension: 0
jealousy: 0
respect: 2
curiosity: 8
resentment: 0

status: короткое описание текущей динамики

memory:
- что персонажи уже заметили друг в друге;
- какие фразы, реакции или поступки запомнили;
- что изменило их отношение.

open_threads:
- обещания;
- договорённости;
- незакрытые вопросы;
- сцены, которые должны вернуться позже.

behavior_next:
- как персонаж должен вести себя в следующей похожей сцене;
- что он может попробовать;
- чего он пока не должен делать.

triggers:
- что может поднять ревность;
- что может поднять доверие;
- что может поднять напряжение;
- что может изменить статус.

last_interaction:
- где и когда была последняя важная сцена;
- чем она закончилась;
- что осталось после неё.
```

## Что означают числа

### affection

Личная тёплая привязанность.

Не равна флирту и не равна интересу.

Низкая `affection` при высокой `curiosity` означает: персонаж зацепился, но ещё не привязался.

### trust

Доверие и ощущение, что на человека можно опереться.

Растёт после поступков, а не после красивых слов.

### tension

Напряжение, конфликтность, неловкость, опасность сорваться.

Может быть даже при симпатии.

### jealousy

Ревность, сравнение внимания, страх быть фоном или потерять позицию.

Не включать без сцены. Но если сцена дала повод — фиксировать.

### respect

Уважение к силе, выдержке, честности, уму, ответственности или стилю поведения.

Растёт, когда персонаж видит что-то значимое.

### curiosity

Интерес, желание понять, проверить, приблизиться или вывести на реакцию.

Высокая `curiosity` не означает любовь.

### resentment

Обида, накопленное раздражение, чувство несправедливости.

Может расти тихо, без открытой ссоры.

## Важные пары

Для основных персонажей relationship memory обязателен:

- `akira__livia_cross`;
- `akira__haru_foster`;
- `akira__raiden_sterling`;
- `livia_cross__haru_foster`;
- `livia_cross__raiden_sterling`;
- `raiden_sterling__haru_foster`;
- `akira__samuel_sterling`;
- `raiden_sterling__samuel_sterling`;
- `akira__ray_carter`;
- `akira__jun_carter`.

NPC получают relationship memory только если они значимы:

- имеют имя;
- появились больше одного раза;
- влияют на сцену;
- конфликтовали с Акирой или другим важным персонажем;
- стали источником слуха;
- связаны с группой, рейтингом, тренировкой, комнатой или будущей линией.

Фоновые NPC не требуют полноценной пары отношений.

## Как обновлять после сцены

После сцены нужно обновлять не всё подряд, а только реальные изменения.

Обновить relationship memory, если:

- была важная фраза;
- появился подкол, который персонаж запомнит;
- кто-то прикрыл другого;
- кто-то задел больную тему;
- появилась договорённость;
- появился повод для ревности;
- вырос интерес;
- возникло уважение;
- появилась обида;
- сцена показала новую привычку или защитную реакцию.

## Поведение по relationship memory

Перед сценой персонаж должен учитывать:

- текущие числа;
- status;
- memory;
- open_threads;
- behavior_next;
- triggers;
- last_interaction.

Если есть `open_threads`, персонаж не должен забывать их без сцены.

Если есть `behavior_next`, оно не является жёстким приказом, но задаёт вероятную манеру следующей реакции.

Если есть `memory`, персонаж должен помнить смысл события, даже если не повторяет фразу дословно.

## Пример

```txt
akira__haru_foster

affection: 0
trust: 0
tension: 0
jealousy: 0
respect: 2
curiosity: 8
resentment: 0

status: первый контакт закреплён; Хару заинтересован спокойной колкостью Акиры и тем, что она не отвечает на обычный флирт.

memory:
- Хару видел Акиру у схемы общежитий и в столовой.
- Хару заметил, что она не суетится и не пытается понравиться.
- Хару заметил её привычку смотреть на пространство как на карту выходов.

open_threads:
- Хару и Акира договорились о баскетболе через неделю.
- Хару хочет понять, что её вообще цепляет.

behavior_next:
- Хару может подходить сам, если есть логичный повод.
- Хару должен помнить прошлый контакт, а не начинать с нуля.
- Хару не влюблён, но ему интересно.

triggers:
- Ревность может появиться, если Акира будет спокойнее с Райденом, чем с Хару.
- Уважение может вырасти, если Хару увидит её контроль в тренировке.
```

## Короткая формула

Score отвечает на вопрос: насколько сильно.

Memory отвечает на вопрос: почему.

Behavior_next отвечает на вопрос: как это сыграть дальше.

Open_threads отвечает на вопрос: что нельзя забыть.
